Overview of the Course Registration System

The Course Registration System is designed to facilitate student registration, course management, and instructor handling within a web-based interface. The system connects to a relational database to store student, course, and instructor data.

The system has a simple user interface made up of HTML forms that interact with the backend via server-side logic. The web interface handles user requests, such as enrolling students, adding instructors, and dropping courses, while the backend processes and stores the data in a database.

Summary of the Contents of Each Page

  • Home Page (index.html): The main landing page with links to other sections of the registration system.
  • Enrollment Page (Enrollment Page.html): A form for students to enter their details and enroll in courses.
  • Add Instructor (addinstructor.html): A page for adding new instructors to the system.
  • Add Course (addcourse.html): A form to add new courses to the system.
  • Register for Courses (registercourse.html): A page where students can select courses to register for.
  • Drop Course (dropcourse.html): A form that allows students to drop a course they are enrolled in.
  • User Manuals (manuels.html): A page linking to both user and programmer manuals.

Functionality: Toggles the visibility of two different sets of navigation links when the respective toggle buttons are clicked.

  • The second button (`menu-toggle2`) toggles the visibility of the second set of navigation links (`nav-links2`).

DOM Manipulation Process

Event Handling: The code listens for the "click" event on both `menu-toggle` and `menu-toggle2`. When clicked, the corresponding `navLinks` or `navlinks2` element will have the `hide` class added or removed, which determines whether the navigation links are visible or hidden.


Enrollment_Input_Validation.js

1. First Name Validation (`FirstName`)

Functionality: Ensures that the first name starts with an uppercase letter followed by 1 to 10 lowercase letters.

Error Message: "First name must start with a capital letter and be followed by 1-10 lowercase letters."

Validation Pattern: ^[A-Z][a-z]{1,10}$ (matches the first character as uppercase and the following 1 to 10 characters as lowercase letters).

2. Last Name Validation (`LastName`)

Functionality: Ensures that the last name starts with an uppercase letter followed by 1 to 10 lowercase letters.

Error Message: "Last name must start with a capital letter and be followed by 1-10 lowercase letters."

Validation Pattern: ^[A-Z][a-z]{1,10}$ (matches the first character as uppercase and the following 1 to 10 characters as lowercase letters).

3. Student Major Validation (`StudentMajor`)

Functionality: Ensures that the student major starts with an uppercase letter, followed by 1 to 15 lowercase letters, optionally followed by a space and an optional hyphen, followed by another uppercase letter and up to 15 lowercase letters.

Error Message: "Student major must be capitalized and follow proper format (e.g., 'Environmental Science')."

Validation Pattern: ^[A-Z][a-z]{1,15}\s?\-?[A-Z]?[a-z]{0,15}$ (matches a capitalized major with optional space and hyphen).

4. Student Email Validation (`StudentEmail`)

Functionality: Ensures that the student email is in a valid format.

Error Message: "Email must follow a conventional format (e.g., 'johnsmith@example.com')."

Validation Pattern: ^[0-9A-Za-z"'-]{0,25}@[A-Za-z\.0-9]+\.(com|org|gov|co|edu|net)$ (matches common email formats like `name@domain.com`).

Validation Process (`Submit-Button` Click Event)

Functionality: Triggers the validation process when the "Submit-Button" is clicked. Each form field is validated asynchronously using promises. The validation checks if the values match their respective regex patterns and ensures that the fields are not empty.

Process:

  • For each form field, a validation check is applied using the corresponding regex pattern.
  • If a field is invalid, its `ValidationData` attribute is set to `false`.
  • After all promises resolve, an alert box is generated that either displays an error message or confirms successful submission.

Alert Box Construction (`AlertBoxConstructor`)

Functionality: Constructs and displays an alert box based on the validation results. If there are any errors, it shows the fields with issues. If all fields are valid, it shows a summary of the correctly submitted data.

Error Message: If any fields are invalid, an alert lists the fields with issues and provides formatting rules for correction.

Success Message: If all fields are valid, an alert displays the submitted data to confirm the form submission.

Example Flow

1. User fills out the form and clicks the submit button.

2. Each field is validated asynchronously using the defined regex patterns.

3. If any fields are invalid, an error message displays the incorrect fields and provides instructions to correct them.

4. If all fields are valid, the form submission is confirmed with an alert displaying the submitted data.

Key Considerations

  • The event listener prevents the form's default submission behavior to allow for custom validation.
  • Each validation is handled asynchronously using promises to ensure that all fields are checked before showing the final alert.
  • The alert box dynamically generates messages based on the validation results.

dropCourseScript.js

Error Message: "First name must contain only letters and spaces."

Validation Pattern: ^[a-zA-Z\s]+$ (matches only alphabetic characters and spaces).

2. Last Name Validation (`studentLastName`)

Functionality: Ensures that the last name contains only letters and spaces.

Error Message: "Last name must contain only letters and spaces."

Validation Pattern: ^[a-zA-Z\s]+$ (matches only alphabetic characters and spaces).

3. Semester Validation (`semester`)

Functionality: Ensures that the semester field is not empty.

Error Message: "Semester is required."

4. Year Validation (`year`)

Functionality: Ensures that the year is a four-digit number and 2025 or later.

Error Message:

  • "Year must be a four-digit number."
  • "Year must be 2025 or later."

Validation Pattern: ^\d{4}$ (matches exactly four digits).

Additional Check: If the entered year is less than 2025, the validation fails.

5. Course Prefix Validation (`coursePrefix`)

Functionality: Ensures that the course prefix is composed of 3 or 4 uppercase letters.

Error Message: "Course prefix must be 3 or 4 capital letters."

Validation Pattern: ^[A-Z]{3,4}$ (matches uppercase alphabetic characters, 3 or 4 characters long).

6. Course Number Validation (`courseNumber`)

Functionality: Ensures that the course number is a three-digit number and not greater than 499.

Error Message:

  • "Course number must be a three-digit number."
  • "Course number cannot be greater than 499."

Validation Pattern: ^\d{3}$ (matches exactly three digits).

Additional Check: If the course number exceeds 499, the validation fails.

7. Course Section Validation (`courseSection`)

Functionality: Ensures that the course section is a two-digit number.

Error Message: "Course section must be a two-digit number."

Validation Pattern: ^\d{2}$ (matches exactly two digits).

Validation and Submission Process

Form Event Listener: The submit event listener is attached to the form, triggering the validation checks when the user attempts to submit.

Error Handling: If invalid input is detected, the form submission is prevented, and an alert with the error messages is shown to the user indicating which fields contain invalid data.

Success Message: If all validations pass, a success message will display the values entered by the user, confirming that the form is ready to be processed.

Functionality: This function dynamically shows or hides the time dropdowns based on the selected day in the `daySelect` dropdown.

function updateTimeDropdown() {\n    for (let key in timeDropdowns) {\n        timeDropdowns[key].hidden = true;\n    }\n    const selectedDay = daySelect.value;\n    if (timeDropdowns[selectedDay]) {\n        timeDropdowns[selectedDay].hidden = false;\n    }\n}

Event Listener: The event listener listens for the `change` event on the `daySelect` dropdown to trigger the update.

if (daySelect) {\n    daySelect.addEventListener("change", updateTimeDropdown);\n    updateTimeDropdown(); // Initialize on load\n}
